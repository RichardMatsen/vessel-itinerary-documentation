      # Project Challenges

      # Data matching challenges

      • Reference mismatches

        • Ports may mismatch in the source and target systems, and require a mapping table to match.
        • Vessels are renamed from time to time, and the names may mismatch in source and target systems.

      • Extraneous data

        • A port call may occur multiple times in the source system, the multiple versions discriminated by an edited date
        • Vessels occur in source that are not used in target
        • Port call dates in source may be outside of the validity dates of the vessel in target system

      • Incomplete data

        • Arrival or Departure may be null
        • Both arrival and departure have a number of columns associated and there is a hierarchy of precedence
          • Actual and scheduled arrival
          • Actual and scheduled departure
        • Actual dates may be empty, in which case fall back to the scheduled dates

      # Process challenges

      • Deleted records

        • The status of an incoming port call may be marked as cancelled, so should be removed in the target system
        • Previously seen port calls may be missing on subsequent runs, so should be removed

      • Manual edits in the target

        • Port calls can be entered or edited in the target, and should not be subsequently overridden by source data
